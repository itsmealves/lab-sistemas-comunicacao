O que são filtros?¶

Filtros são componentes que são capazes de remover componentes indesejados ou evidenciar características de um sinal. Em sistemas de comunicação, são normalmente utilizados para separar componentes de informação ou corrigir distorções, por exemplo. Um filtro de linhas transversais funciona pela convolução do sinal de entrada com seus coeficientes, ou seja, as amostras do sinal de entrada são colocadas, uma a uma, em linhas de atraso ponderadas e seus resultados são somados.

Vamos ver na aula de hoje como podemos implementar filtros (utilizando python) e utilizá-los para remover dados indesejados de sinais.

Sinais¶

Antes de estudarmos diretamente com filtros, vamos primeiro entender como podemos representar sinais. Assim, poderemos criar diferentes sinais para filtrarmos mais adiante.

Amostragem¶

Os sinais não podem ser representados como sinais contínuos dentro de um computador, pois simplesmente não existe memória infinita para representar o domínio (e.g. tempo) sem antes definir uma maneira de tornar este discreto. Existe, no entanto, uma forma simples de pensar nisso. Amostrar um sinal é representar este como uma sequência de amostras espaçadas igualmente. Desta forma, desde que o formato de onda seja mantido, as mesmas informações que estão presentes no sinal de domínio contínuo também estarão presentes no sinal de domínio discreto. Isto pode ser visto na Figura 2. A representação matemática da amostragem é a multiplicação de um sinal por um trem de impulsos separados entre si por um multiplo de uma constante (i.e. o período de amostragem).

Teorema da amostragem¶

Para que o formato de onda de um sinal após a coleta das amostras, existe um critério que é definido pelo chamado teorema da amostragem que indica a frequência mínima (ou período máximo) do trem de impulsos para que não ocorra perda de informação durante este processo. O critério define que a taxa mínima de amostragem deve ser igual a frequência máxima presente no sinal. Podemos assim citar alguns exemplos para demonstrar a aplicabilidade deste critério:

  • Uma chamada telefônica tem como taxa de amostragem 8kHz, pois a voz humana muito dificilmente pode superar a marca de 4kHz. Logo, a frequência mínima de amostragem é de 8kHz.
  • A taxa de amostragem de um CD de áudio é de 44.1kHz, pois a frequência máxima que o ouvido de um ser humano pode ouvir é de 20kHz, então, a taxa de amostragem mínima para poder representar todo a banda audível é de 40kHz.

# Function to plot signal data
# Source: https://matplotlib.org/api/_as_gen/matplotlib.pyplot.plot.html
def plot_signal(figure, title, labels, data):
    plt.figure(figure)
    plt.title(title)
    plt.xlabel(labels[0])
    plt.ylabel(labels[1])
    plt.plot(data[0], data[1])

# Function to plot data in frequency domain
# Source: https://www.mathworks.com/help/matlab/ref/fft.html
def plot_frequency(figure, data, sampling_rate):
    s = np.fft.fft(data)
    data_length = data.shape[0]
    p2 = np.abs(s / data_length)
    p1 = p2[0:int(round(data_length / 2))]
    p1[1:] = 2 * p1[1:];
    domain = sampling_rate * np.arange(0, data_length / 2) / data_length
    
    plt.figure(figure)
    plt.plot(domain, p1)

# Function to plot filter response
# Source: https://docs.scipy.org/doc/scipy/reference/generated/scipy.signal.freqz.html
def plot_filter_response(figure, f):
    plt.figure(figure)
    w, h = signal.freqz(f)

    fig, ax1 = plt.subplots()
    ax1.set_title('Digital filter frequency response')

    ax1.plot(w, 20 * np.log10(abs(h)), 'b')
    ax1.set_ylabel('Amplitude [dB]', color='b')
    ax1.set_xlabel('Frequency [rad/sample]')

    ax2 = ax1.twinx()
    angles = np.unwrap(np.angle(h))
    ax2.plot(w, angles, 'g')
    ax2.set_ylabel('Angle (radians)', color='g')
    ax2.grid()
    ax2.axis('tight')

Filtragem¶

Vamos agora ver como é realizado o processo de filtragem. Vamos fazer isto nos seguintes tópicos:

Depois vamos ver em um exemplo prático a filtragem de sinais acústicos.

Representação de um filtro¶

Um filtro pode ser representado pela coleção de seus coeficientes, que são os elementos que vão interagir com o sinal pelo processo de convolução. Vamos usar o scipy para gerar coeficientes de filtros FIR utilizando o método de janelamento e vamos ver os efeitos que estes causam tanto na frequência como na fase. Vamos falar sobre filtros FIR e IIR em breve.

In [5]:
low_pass_filter = signal.firwin(151, 125.0 / (sampling_rate / 2.0))
high_pass_filter = signal.firwin(151, 125.0 / (sampling_rate / 2.0), pass_zero=False)

plot_filter_response(3, low_pass_filter)
plot_filter_response(4, high_pass_filter)

<Figure size 432x288 with 0 Axes>

Podemos ver que, para ambos os filtros, existem duas regiões:

  • Existe uma região que não causa distorção no sinal. Esta região representa a banda em que as informações do sinal não são perdidas. No primeiro filtro (que é um passa-baixa), por exemplo, essa região abrange as frequências mais baixas.
  • A outra região envolve um fenômeno de distorção que se comporta como um atenuador de frequências. Nesta banda de frequências, as informações são atenuadas ou, idealmente, completamente perdidas.

Mas por quê isso acontece? Se visualizarmos (como faremos a seguir) os coeficientes dos filtros, vamos perceber que eles se aproximam muito da função $sinc(t)$, que tem como resposta no domínio da frequência a função degrau! Ou seja, a convolução de um sinal com a função $sinc(t)$ equivale a multiplicar pelo degrau no domínio na frequência. Essa multiplicação no domínio da frequência causa uma região $K$ em que o sinal não é distorcido, onde $K = \{w | step(w) = 1\}$ e a região de atenuação $L$, onde $L = \{w | step(w) = 0\}$.

In [6]:
low_pass_domain = np.arange(0, low_pass_filter.shape[0])
plot_signal(1, 'Low pass filter', ('Coefficients', ''), (low_pass_domain, low_pass_filter))
plot_frequency(2, low_pass_filter, sampling_rate)

high_pass_domain = np.arange(0, high_pass_filter.shape[0])
plot_signal(3, 'High pass filter', ('Coefficients', ''), (high_pass_domain, high_pass_filter))
plot_frequency(4, high_pass_filter, sampling_rate)

Na prática, é impossível representar fielmente a função $sinc(t)$ porque isso demandaria recursos infinitos e não causalidade, o que não é factível. Por isso, quando vemos a resposta do filtro no domínio da frequência, a função degrau também não é perfeita, sendo uma aproximação cuja derivada na região de maior variação é finita.

No entanto, essa aproximação pode ser boa o suficiente de acordo com a ordem do filtro (a ordem $N$ de um filtro é igual ao número de coeficientes mais um), pois a medida que o número de coeficientes cresce, mais próxima da função degrau a resposta do filtro na frequência se aproxima.

E para aplicarmos um filtro? Veremos que a operação adequada no domínio da frequência é a multiplicação como sugerem as figuras abaixo. Ou seja, no domínio do tempo, precisamos de uma convolução!

In [7]:
plot_frequency(1, s3 / 46, sampling_rate)
plot_frequency(1, low_pass_filter, sampling_rate)

plot_frequency(2, s3 / 46, sampling_rate)
plot_frequency(2, high_pass_filter, sampling_rate)

Obs.: FIR vs IIR¶

Os filtros FIR não possuem malha de realimentação, ao contrário dos filtros IIR. Isto traz diversas consequências positivas e negativas para a utilização de cada uma das estruturas. Na nossa implementação, a única diferença é que o filtro FIR possui apenas uma coleção de coeficientes, enquanto o filtro IIR tem duas coleções (coeficientes do numerador e do denominador). Por questão de simplicidade, vamos focar nos filtros FIR, mas recomendo uma leitura sobre o conteúdo (no final deste material, tem uma seção de leituras relacionadas que inclui um link para um material sobre este assunto).

Convolução¶

A operação de convolução pode ser definida de acordo com a equação abaixo. Esta pode ser interpretada como a projeção da forma de uma onda na outra, a medida que uma destas é deslocada no domínio. Quando o sinal está no domínio do tempo, fazemos com que as amostras sejam projetadas depois de linhas de atraso, assim como apresentado na Figura 1.

$$s(t)*h(t) = \int_{-\infty}^{\infty} s(\tau)h(t - \tau)d\tau$$

Com o exercício anterior, podemos implementar uma função para executar a convolução. A seguir, temos uma possível implementação.

In [8]:
def convolve(x, h):
    xr = [0] * h.shape[0]
    
    # Fills the regressor with signal and computes dot products for each sample
    def run(signal, regressor):
        y = []
        for sample in signal:
            regressor.insert(0, sample)
            regressor.pop()
            y.append(np.dot(np.array(regressor), h))
            
        return y
    
    # a is filling the regressor with our signal
    # and b is filling the regressor with zeros
    a = run(x, xr)
    b = run(np.zeros(h.shape[0] - 1), xr)
    
    y = np.array(a + b)
    return y

Agora que conhecemos com o que vamos trabalhar, podemos começar a produzir nossa solução. Se utilizarmos um instrumento de medição, veremos que o ruído agudo é uma onda senoidal de 8kHz. Sabemos também que chamadas telefônicas são amostradas a 8kHz, logo, nunca vamos encontrar frequências maiores do que 4kHz. Portanto, se utilizarmos um filtro passa-baixa de 6kHz, o ruído será eliminado com certa folga.

In [11]:
# Input
audio_sampling, stereo_data = wavfile.read('static/audio/noisy_mom.wav')
num_samples = stereo_data.shape[0]
num_channels = stereo_data.shape[1]
audio_duration = num_samples / audio_sampling   # seconds
max_value = np.abs(stereo_data).max()

# Preprocessing
normalized_stereo_data = stereo_data / max_value

# Filtering
channels = []
f = signal.firwin(500, 6000.0 / (audio_sampling / 2.0))

for channel in range(num_channels):
    labels = ('Time (seconds)', 'Amplitude')
    y = convolve(normalized_stereo_data[:, channel], f)
    domain = np.linspace(0, y.shape[0] * 1.0 / audio_sampling, y.shape[0])
    
    channels.append(y)
    plot_signal(channel + 1, 'Result ' + str(channel + 1), labels, (domain, y))

# Output preparation
normalized_data = np.stack(channels, axis=1)
data = (normalized_data * max_value).astype(stereo_data.dtype)

# Output recording
wavfile.write('static/audio/filtered_mom.wav', audio_sampling, data)

Podemos ver que o padrão que causava ruído não está mais presente no sinal! Isto pode ser confirmado pelo arquivo de audio que foi produzido. Tome um tempo para checar o resultado. Para finalizar, vamos ver a resposta em frequência e fase do filtro que foi utilizado.
