O que são filtros?¶

Filtros so componentes que so capazes de remover componentes indesejados ou evidenciar caractersticas de um sinal. Em sistemas de comunicao, so normalmente utilizados para separar componentes de informao ou corrigir distores, por exemplo. Um filtro de linhas transversais funciona pela convoluo do sinal de entrada com seus coeficientes, ou seja, as amostras do sinal de entrada so colocadas, uma a uma, em linhas de atraso ponderadas e seus resultados so somados.

Vamos ver na aula de hoje como podemos implementar filtros (utilizando python) e utiliz-los para remover dados indesejados de sinais.

Sinais¶

Antes de estudarmos diretamente com filtros, vamos primeiro entender como podemos representar sinais. Assim, poderemos criar diferentes sinais para filtrarmos mais adiante.

Amostragem¶

Os sinais no podem ser representados como sinais contnuos dentro de um computador, pois simplesmente no existe memria infinita para representar o domnio (e.g. tempo) sem antes definir uma maneira de tornar este discreto. Existe, no entanto, uma forma simples de pensar nisso. Amostrar um sinal representar este como uma sequncia de amostras espaadas igualmente. Desta forma, desde que o formato de onda seja mantido, as mesmas informaes que esto presentes no sinal de domnio contnuo tambm estaro presentes no sinal de domnio discreto. Isto pode ser visto na Figura 2. A representao matemtica da amostragem a multiplicao de um sinal por um trem de impulsos separados entre si por um multiplo de uma constante (i.e. o perodo de amostragem).

Teorema da amostragem¶

Para que o formato de onda de um sinal aps a coleta das amostras, existe um critrio que definido pelo chamado teorema da amostragem que indica a frequncia mnima (ou perodo mximo) do trem de impulsos para que no ocorra perda de informao durante este processo. O critrio define que a taxa mnima de amostragem deve ser igual a frequncia mxima presente no sinal. Podemos assim citar alguns exemplos para demonstrar a aplicabilidade deste critrio:

  • Uma chamada telefnica tem como taxa de amostragem 8kHz, pois a voz humana muito dificilmente pode superar a marca de 4kHz. Logo, a frequncia mnima de amostragem de 8kHz.
  • A taxa de amostragem de um CD de udio de 44.1kHz, pois a frequncia mxima que o ouvido de um ser humano pode ouvir de 20kHz, ento, a taxa de amostragem mnima para poder representar todo a banda audvel de 40kHz.

# Function to plot signal data
# Source: https://matplotlib.org/api/_as_gen/matplotlib.pyplot.plot.html
def plot_signal(figure, title, labels, data):
    plt.figure(figure)
    plt.title(title)
    plt.xlabel(labels[0])
    plt.ylabel(labels[1])
    plt.plot(data[0], data[1])

# Function to plot data in frequency domain
# Source: https://www.mathworks.com/help/matlab/ref/fft.html
def plot_frequency(figure, data, sampling_rate):
    s = np.fft.fft(data)
    data_length = data.shape[0]
    p2 = np.abs(s / data_length)
    p1 = p2[0:int(round(data_length / 2))]
    p1[1:] = 2 * p1[1:];
    domain = sampling_rate * np.arange(0, data_length / 2) / data_length
    
    plt.figure(figure)
    plt.plot(domain, p1)

# Function to plot filter response
# Source: https://docs.scipy.org/doc/scipy/reference/generated/scipy.signal.freqz.html
def plot_filter_response(figure, f):
    plt.figure(figure)
    w, h = signal.freqz(f)

    fig, ax1 = plt.subplots()
    ax1.set_title('Digital filter frequency response')

    ax1.plot(w, 20 * np.log10(abs(h)), 'b')
    ax1.set_ylabel('Amplitude [dB]', color='b')
    ax1.set_xlabel('Frequency [rad/sample]')

    ax2 = ax1.twinx()
    angles = np.unwrap(np.angle(h))
    ax2.plot(w, angles, 'g')
    ax2.set_ylabel('Angle (radians)', color='g')
    ax2.grid()
    ax2.axis('tight')

Representação de um filtro¶

Um filtro pode ser representado pela coleo de seus coeficientes, que so os elementos que vo interagir com o sinal pelo processo de convoluo. Vamos usar o scipy para gerar coeficientes de filtros FIR utilizando o mtodo de janelamento e vamos ver os efeitos que estes causam tanto na frequncia como na fase. Vamos falar sobre filtros FIR e IIR em breve.

low_pass_filter = signal.firwin(151, 125.0 / (sampling_rate / 2.0))
high_pass_filter = signal.firwin(151, 125.0 / (sampling_rate / 2.0), pass_zero=False)

plot_filter_response(3, low_pass_filter)
plot_filter_response(4, high_pass_filter)

<Figure size 432x288 with 0 Axes>

Podemos ver que, para ambos os filtros, existem duas regies:

  • Existe uma regio que no causa distoro no sinal. Esta regio representa a banda em que as informaes do sinal no so perdidas. No primeiro filtro (que um passa-baixa), por exemplo, essa regio abrange as frequncias mais baixas.
  • A outra regio envolve um fenmeno de distoro que se comporta como um atenuador de frequncias. Nesta banda de frequncias, as informaes so atenuadas ou, idealmente, completamente perdidas.

Mas por qu isso acontece? Se visualizarmos (como faremos a seguir) os coeficientes dos filtros, vamos perceber que eles se aproximam muito da funo $sinc(t)$, que tem como resposta no domnio da frequncia a funo degrau! Ou seja, a convoluo de um sinal com a funo $sinc(t)$ equivale a multiplicar pelo degrau no domnio na frequncia. Essa multiplicao no domnio da frequncia causa uma regio $K$ em que o sinal no distorcido, onde $K = \{w | step(w) = 1\}$ e a regio de atenuao $L$, onde $L = \{w | step(w) = 0\}$.

plot_signal(1, 'Low pass filter', ('Coefficients', ''), (low_pass_domain, low_pass_filter))

high_pass_domain = np.arange(0, high_pass_filter.shape[0])

Na prtica, impossvel representar fielmente a funo $sinc(t)$ porque isso demandaria recursos infinitos e no causalidade, o que no factvel. Por isso, quando vemos a resposta do filtro no domnio da frequncia, a funo degrau tambm no perfeita, sendo uma aproximao cuja derivada na regio de maior variao finita.

No entanto, essa aproximao pode ser boa o suficiente de acordo com a ordem do filtro (a ordem $N$ de um filtro igual ao nmero de coeficientes mais um), pois a medida que o nmero de coeficientes cresce, mais prxima da funo degrau a resposta do filtro na frequncia se aproxima.

E para aplicarmos um filtro? Veremos que a operao adequada no domnio da frequncia a multiplicao como sugerem as figuras abaixo. Ou seja, no domnio do tempo, precisamos de uma convoluo!

Obs.: FIR vs IIR¶

Os filtros FIR no possuem malha de realimentao, ao contrrio dos filtros IIR. Isto traz diversas consequncias positivas e negativas para a utilizao de cada uma das estruturas. Na nossa implementao, a nica diferena que o filtro FIR possui apenas uma coleo de coeficientes, enquanto o filtro IIR tem duas colees (coeficientes do numerador e do denominador). Por questo de simplicidade, vamos focar nos filtros FIR, mas recomendo uma leitura sobre o contedo (no final deste material, tem uma seo de leituras relacionadas que inclui um link para um material sobre este assunto).

Convolução¶

A operao de convoluo pode ser definida de acordo com a equao abaixo. Esta pode ser interpretada como a projeo da forma de uma onda na outra, a medida que uma destas deslocada no domnio. Quando o sinal est no domnio do tempo, fazemos com que as amostras sejam projetadas depois de linhas de atraso, assim como apresentado na Figura 1.

$$s(t)*h(t) = \int_{-\infty}^{\infty} s(\tau)h(t - \tau)d\tau$$

In [8]:
def convolve(x, h):
    xr = [0] * h.shape[0]
    
    # Fills the regressor with signal and computes dot products for each sample
    def run(signal, regressor):
        y = []
        for sample in signal:
            regressor.insert(0, sample)
            regressor.pop()
            y.append(np.dot(np.array(regressor), h))
            
        return y
    
    # a is filling the regressor with our signal
    # and b is filling the regressor with zeros
    a = run(x, xr)
    b = run(np.zeros(h.shape[0] - 1), xr)
    
    y = np.array(a + b)
    return y

Agora que conhecemos com o que vamos trabalhar, podemos comear a produzir nossa soluo. Se utilizarmos um instrumento de medio, veremos que o rudo agudo uma onda senoidal de 8kHz. Sabemos tambm que chamadas telefnicas so amostradas a 8kHz, logo, nunca vamos encontrar frequncias maiores do que 4kHz. Portanto, se utilizarmos um filtro passa-baixa de 6kHz, o rudo ser eliminado com certa folga.

In [11]:
# Input
audio_sampling, stereo_data = wavfile.read('static/audio/noisy_mom.wav')
num_samples = stereo_data.shape[0]
num_channels = stereo_data.shape[1]
audio_duration = num_samples / audio_sampling   # seconds
max_value = np.abs(stereo_data).max()

# Preprocessing
normalized_stereo_data = stereo_data / max_value

# Filtering
channels = []
f = signal.firwin(500, 6000.0 / (audio_sampling / 2.0))

for channel in range(num_channels):
    labels = ('Time (seconds)', 'Amplitude')
    y = convolve(normalized_stereo_data[:, channel], f)
    domain = np.linspace(0, y.shape[0] * 1.0 / audio_sampling, y.shape[0])
    
    channels.append(y)
    plot_signal(channel + 1, 'Result ' + str(channel + 1), labels, (domain, y))

# Output preparation
normalized_data = np.stack(channels, axis=1)
data = (normalized_data * max_value).astype(stereo_data.dtype)

# Output recording
wavfile.write('static/audio/filtered_mom.wav', audio_sampling, data)
